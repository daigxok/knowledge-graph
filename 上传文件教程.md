# 📤 上传文件到 GitHub - 超简单教程

## 最简单的方法 - 使用 GitHub Desktop

---

## 📥 第一步：下载 GitHub Desktop

### 1. 打开浏览器
访问：https://desktop.github.com/

### 2. 下载
点击 **"Download for Windows"** 按钮

### 3. 安装
- 双击下载的文件
- 等待安装完成（自动安装，无需选择）
- 安装完成后会自动打开

### 4. 登录
- 点击 **"Sign in to GitHub.com"**
- 输入你的 GitHub 用户名和密码
- 点击 **"Sign in"**

✅ 完成！GitHub Desktop 已准备好

---

## 📂 第二步：获取仓库到本地

### 1. 克隆仓库
在 GitHub Desktop 中：
- 点击左上角 **File** → **Clone repository**
- 或者点击中间的 **"Clone a repository from the Internet..."**

### 2. 选择仓库
- 在列表中找到 **daigxok/knowledge-graph**
- 如果看不到，点击 **URL** 标签，输入：
  ```
  https://github.com/daigxok/knowledge-graph
  ```

### 3. 选择保存位置
- **Local path**: 选择保存位置（例如：`D:\Projects\knowledge-graph`）
- 记住这个位置！

### 4. 点击 Clone
- 点击蓝色的 **Clone** 按钮
- 等待下载完成（可能需要几分钟）

✅ 完成！你现在有了仓库的本地副本

---

## 📋 第三步：复制新文件

### 1. 打开两个文件夹

**文件夹 A**：你当前的项目文件夹
- 包含所有新文件（landing.html, auth.html 等）

**文件夹 B**：GitHub Desktop 克隆的文件夹
- 就是第二步中选择的位置
- 例如：`D:\Projects\knowledge-graph`

### 2. 全选文件夹 A 的所有内容
- 在文件夹 A 中按 **Ctrl+A**（全选）
- 按 **Ctrl+C**（复制）

### 3. 粘贴到文件夹 B
- 打开文件夹 B
- 按 **Ctrl+V**（粘贴）
- 弹出提示时，选择 **"替换目标中的文件"**
- 勾选 **"对所有当前项目执行此操作"**
- 点击 **"是"** 或 **"替换"**

### 4. 等待复制完成
- 可能需要几分钟
- 不要关闭窗口

✅ 完成！所有文件已复制到仓库文件夹

---

## ✅ 第四步：提交更改

### 1. 回到 GitHub Desktop
- 你会看到左侧列出了很多文件
- 这些都是新增或修改的文件

### 2. 查看更改（可选）
- 点击左侧的文件名可以查看具体更改
- 绿色 = 新增内容
- 红色 = 删除内容

### 3. 填写提交信息
在左下角找到两个输入框：

**Summary（必填）**：
```
更新到 v2.0 - 添加登录注册系统和首页
```

**Description（可选）**：
```
- 新增用户认证系统
- 新增精美首页
- 新增完整使用文档
- 优化用户体验
```

### 4. 提交
- 点击蓝色的 **"Commit to main"** 按钮
- 等待几秒钟

✅ 完成！更改已提交到本地仓库

---

## 🚀 第五步：上传到 GitHub

### 1. 推送更改
- 在 GitHub Desktop 右上角
- 点击 **"Push origin"** 按钮
- 或者按 **Ctrl+P**

### 2. 等待上传
- 会显示上传进度
- 可能需要几分钟（取决于文件大小和网速）
- 不要关闭 GitHub Desktop

### 3. 上传完成
- 当按钮变回 **"Fetch origin"** 时，说明上传完成
- 或者显示 **"Last fetched just now"**

✅ 完成！文件已上传到 GitHub

---

## 🌐 第六步：启用网站（首次需要）

### 1. 访问 GitHub 仓库
打开浏览器，访问：
```
https://github.com/daigxok/knowledge-graph
```

### 2. 进入设置
- 点击页面右上角的 **Settings**（设置）

### 3. 找到 Pages
- 在左侧菜单中，向下滚动
- 找到并点击 **Pages**

### 4. 配置 Pages
- 在 **Source** 下拉菜单中
- 选择 **main** 分支
- 保持 **/ (root)** 不变
- 点击 **Save** 按钮

### 5. 等待部署
- 页面会显示：
  ```
  Your site is ready to be published at 
  https://daigxok.github.io/knowledge-graph/
  ```
- 等待 2-5 分钟

✅ 完成！网站已启用

---

## 🎉 第七步：访问你的网站

### 等待几分钟后，访问：

```
https://daigxok.github.io/knowledge-graph/landing.html
```

你应该能看到新的首页！

## ⚠️ 常见问题

### Q1: 找不到 "Clone repository" 选项？
**A**: 点击左上角的 **File** 菜单，就能看到了。

### Q2: 列表中没有我的仓库？
**A**: 
1. 确认已登录 GitHub Desktop
2. 点击 **URL** 标签
3. 输入仓库地址：`https://github.com/daigxok/knowledge-graph`

### Q3: 复制文件时提示权限错误？
**A**: 
1. 关闭 GitHub Desktop
2. 以管理员身份运行
3. 重新复制文件

### Q4: "Push origin" 按钮是灰色的？
**A**: 
1. 确认已经点击了 "Commit to main"
2. 等待几秒钟
3. 如果还是灰色，说明没有需要上传的更改

### Q5: 上传很慢或失败？
**A**: 
1. 检查网络连接
2. 尝试使用 VPN
3. 等待一段时间后重试

### Q6: 网站显示 404？
**A**: 
1. 确认 GitHub Pages 已启用
2. 等待 5-10 分钟
3. 清除浏览器缓存（Ctrl+Shift+Delete）
4. 确认访问的 URL 正确

### Q7: 网站显示旧内容？
**A**: 
1. 清除浏览器缓存
2. 按 Ctrl+F5 强制刷新
3. 等待几分钟让 GitHub 更新

## 💡 小贴士

### 提高上传速度
- 使用稳定的网络
- 避免在网络高峰期上传
- 如果文件很大，可以分批上传

### 避免错误
- 不要在上传过程中关闭 GitHub Desktop
- 不要同时在多个地方修改文件
- 定期备份重要文件

### 保持整洁
- 每次提交都写清楚做了什么
- 不要上传临时文件和缓存
- 定期清理不需要的文件
