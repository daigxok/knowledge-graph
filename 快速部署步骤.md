# ⚡ 快速部署步骤

## 最简单的方法 - 使用 GitHub Desktop

---

## 📥 第一步：下载 GitHub Desktop

访问 https://desktop.github.com/ 下载并安装

---

## 🔑 第二步：登录

打开 GitHub Desktop，登录你的 GitHub 账号

---

## 📦 第三步：克隆仓库

1. 点击 **File** → **Clone repository**
2. 选择 `daigxok/knowledge-graph`
3. 选择保存位置（例如：`D:\Projects\knowledge-graph`）
4. 点击 **Clone**

---

## 📁 第四步：复制文件

1. 打开你当前的项目文件夹（包含所有新文件）
2. **全选所有文件和文件夹**（Ctrl+A）
3. **复制**（Ctrl+C）
4. 打开 GitHub Desktop 克隆的文件夹
5. **粘贴**（Ctrl+V）
6. 选择 **替换所有文件**

---

## ✅ 第五步：提交更改

回到 GitHub Desktop：

1. 左下角 **Summary** 框输入：
   ```
   更新到 v2.0 - 添加登录系统和首页
   ```

2. 点击 **Commit to main**

---

## 🚀 第六步：推送到 GitHub

点击右上角的 **Push origin** 按钮，等待上传完成

---

## 🌐 第七步：启用 GitHub Pages

1. 访问 https://github.com/daigxok/knowledge-graph
2. 点击 **Settings**
3. 左侧菜单点击 **Pages**
4. **Source** 选择 **main** 分支
5. 点击 **Save**

---

## 🎉 完成！

等待 2-5 分钟后，访问：

**https://daigxok.github.io/knowledge-graph/landing.html**

---

## ⚠️ 注意事项

1. **确保所有文件都复制了**，包括子文件夹
2. **不要删除** `.git` 文件夹（隐藏文件夹）
3. **等待 2-5 分钟** GitHub Pages 才会生效
4. 如果看不到更新，**清除浏览器缓存**（Ctrl+Shift+Delete）

## 🆘 遇到问题？

### 问题1：GitHub Desktop 显示很多文件
**正常！** 因为添加了新功能，直接提交即可

### 问题2：推送失败
**解决**：确认已登录 GitHub Desktop

### 问题3：网站显示 404
**解决**：
1. 确认 Pages 已启用
2. 等待 5-10 分钟
3. 清除浏览器缓存

### 问题4：功能不正常
**解决**：
1. 按 F12 查看控制台错误
2. 确认所有文件都已上传
3. 清除浏览器缓存
