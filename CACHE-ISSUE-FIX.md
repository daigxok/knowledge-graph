# 🔧 浏览器缓存问题解决方案

## 问题描述

如果你在浏览器中看到的节点数是 **25个** 而不是 **75个**，这是因为浏览器缓存了旧的数据文件。

---

## ✅ 解决方法

### 方法1: 强制刷新（推荐）⭐

**Windows/Linux:**
```
按 Ctrl + F5
或
按 Ctrl + Shift + R
```

**Mac:**
```
按 Cmd + Shift + R
```

### 方法2: 清除浏览器缓存

1. 打开浏览器开发者工具 (F12)
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 方法3: 使用测试页面

打开这个页面查看真实数据（会绕过缓存）:
```
knowledge-graph/test-fresh-load.html
```

### 方法4: 使用无痕/隐私模式

在浏览器的无痕模式下打开 `index.html`

---

## 📊 正确的数据

系统应该显示:
- **总节点数**: 75个
- **总边关系**: 154条
- **原始节点**: 25个
- **Phase 1新增**: 50个

---

## 🧪 验证数据

### 快速验证
打开浏览器控制台 (F12)，运行:
```javascript
fetch('data/nodes.json?t=' + Date.now())
  .then(r => r.json())
  .then(d => console.log('节点数:', d.nodes.length))
```

应该显示: `节点数: 75`

### 使用测试页面
1. 打开 `test-fresh-load.html`
2. 查看显示的节点数
3. 如果显示75个，说明数据正确

---

## 🔍 为什么会有缓存问题？

浏览器会缓存JSON文件以提高性能。当数据文件更新后，浏览器可能仍然使用旧的缓存版本。

**缓存的文件:**
- `data/nodes.json` (旧版本: 25个节点)
- `data/edges.json` (旧版本: 55条边)

**更新后的文件:**
- `data/nodes.json` (新版本: 75个节点)
- `data/edges.json` (新版本: 154条边)
