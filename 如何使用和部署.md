## 第二部分：如何部署到网站

### 🌐 为什么要部署？

部署后可以：
- ✅ 通过网址访问（不用打开文件）
- ✅ 分享给其他人使用
- ✅ 在任何设备上访问
- ✅ 看起来更专业

### ⚡ 最简单的部署方法

#### 使用 GitHub Desktop（推荐）

**第一步：安装**
1. 访问 https://desktop.github.com/
2. 下载并安装
3. 登录 GitHub 账号

**第二步：克隆仓库**
1. File → Clone repository
2. 选择 `daigxok/knowledge-graph`
3. 选择保存位置
4. 点击 Clone

**第三步：复制文件**
1. 全选你的项目文件（Ctrl+A）
2. 复制（Ctrl+C）
3. 粘贴到克隆的文件夹（Ctrl+V）
4. 替换所有文件

**第四步：提交**
1. 回到 GitHub Desktop
2. 输入提交信息：`更新到 v2.0`
3. 点击 Commit to main

**第五步：推送**
1. 点击 Push origin
2. 等待上传完成

**第六步：启用 Pages**
1. 访问 https://github.com/daigxok/knowledge-graph
2. Settings → Pages
3. Source 选择 main
4. 点击 Save

**完成！**
等待 2-5 分钟后访问：
```
https://daigxok.github.io/knowledge-graph/landing.html
```

## 第三部分：常见问题

### 使用相关

**Q: 忘记密码怎么办？**
A: 清除浏览器数据重新注册（本地存储）

**Q: 学习数据会丢失吗？**
A: 定期点击"导出学习数据"备份

**Q: 可以在手机上用吗？**
A: 可以！系统支持移动端

**Q: 如何查看所有节点？**
A: 点击"清除所有筛选"按钮

### 部署相关

**Q: 必须部署到 GitHub 吗？**
A: 不是，也可以部署到 Netlify、Vercel 等

**Q: 部署需要多长时间？**
A: 首次约 20-25 分钟，后续更新 5 分钟

**Q: 部署后如何更新？**
A: 修改文件 → GitHub Desktop 提交 → 推送

**Q: 网站显示 404？**
A: 等待 5-10 分钟，清除浏览器缓存
