# 📄 一页纸上传指南

## 5分钟上传文件到 GitHub

---

## 🎯 目标
将你的项目文件上传到：`https://github.com/daigxok/knowledge-graph`

---

## 📋 准备工作

✅ 你有 GitHub 账号  
✅ 你有仓库：`daigxok/knowledge-graph`  
✅ 你的项目文件都在一个文件夹里

---

## 🚀 5个步骤

### 1️⃣ 下载 GitHub Desktop（5分钟）
- 访问：https://desktop.github.com/
- 下载 → 安装 → 登录

### 2️⃣ 克隆仓库（2分钟）
- File → Clone repository
- 选择：`daigxok/knowledge-graph`
- 保存位置：`D:\Projects\knowledge-graph`
- 点击 Clone

### 3️⃣ 复制文件（2分钟）
- 打开你的项目文件夹
- 全选（Ctrl+A）→ 复制（Ctrl+C）
- 打开克隆的文件夹（`D:\Projects\knowledge-graph`）
- 粘贴（Ctrl+V）→ 替换所有

### 4️⃣ 提交（1分钟）
- 回到 GitHub Desktop
- 左下角输入：`更新到 v2.0`
- 点击：**Commit to main**

### 5️⃣ 上传（3分钟）
- 点击右上角：**Push origin**
- 等待上传完成

---

## 🌐 启用网站（首次，2分钟）

1. 访问：https://github.com/daigxok/knowledge-graph
2. Settings → Pages
3. Source 选择：**main**
4. 点击：**Save**

---

## ✅ 完成！

等待 2-5 分钟后访问：

```
https://daigxok.github.io/knowledge-graph/landing.html
```

---

## 🔄 以后更新（3分钟）

```
修改文件 → GitHub Desktop → 提交 → 推送 → 完成
```

---

## ⚠️ 常见问题

| 问题 | 解决 |
|------|------|
| 找不到仓库 | 点击 URL 标签，输入仓库地址 |
| 上传失败 | 检查网络，重试 |
| 网站 404 | 等待 5-10 分钟，清除缓存 |
| 显示旧内容 | Ctrl+F5 强制刷新 |
