## 🎓 高等数学学域知识图谱：教师使用手册

**版本**: 1.0  
**适用对象**: 高等数学任课教师 / 教研组  

---

## 一、项目可以解决什么问题？

- **从“章节目录”走向“学域结构”**  
  帮助教师从五大学域视角重新审视课程结构，清楚看到：某一章内容分别服务于“变化与逼近 / 结构与累积 / 优化与决策 / 不确定性处理 / 真实问题建模”中的哪一块。

- **从“单点知识”走向“依赖关系”**  
  通过图谱中的前置依赖、跨域连边，快速识别：  
  - 当前要讲的知识点，**真正需要补哪几个前置**；  
  - 哪些内容可以整合成一节“主题课例”来讲。

- **从“题海练习”走向“技能导向”**  
  14 个学生侧 Skills（含 Phase2 深度内容）为每个核心知识点准备了：  
  - **进阶主题**（advancedTopics）  
  - **进阶练习**（advancedExercises：题目 + 提示 + 解答关键步骤 + 要点）  
  - **项目任务**（projects）  
  教师可以据此设计分层作业、小测、项目学习任务。

- **从“个人经验”走向“可复用教案框架”**  
  `limit-conflict-teaching.html`、`teacher-agents.html`、`student-skills.html` 等页面提供了：  
  - 认知冲突课例（极限、采样定理）；  
  - 教师侧 7 大 AI Agents 分工；  
  - 学生 Skills 总览卡片。  
  这些都可以作为教研组共享的“可复用模板”。

## 二、快速上手：教师的一天怎么用？

### 1. 课前 30–60 分钟：备课规划

1. **打开主图 `index.html`**
   - 在浏览器打开 `knowledge-graph/index.html`。  
   - 利用顶部“**学域徽章**”和左侧“**学域概览**”：  
     - 勾选本节课将涉及的学域（如讲极限时，一般在“变化与逼近”主域内）。  
     - 观察该学域下已有多少节点、核心思想简述。

2. **确定本节课的“目标节点”与“前置节点”**
   - 在图中点击计划要讲的节点（如 `极限的定义`、`函数的连续性`、`傅里叶级数应用` 等）。  
   - 在右侧“节点详情”中，重点看：
     - **📚 前置知识**：本节课前要先确保哪些知识已经掌握？  
     - **🎯 所属学域**：是否需要提醒学生“这是在学域一中衔接后续内容”？  
   - 由此决定：  
     - 是否在上课前 5–10 分钟补一个“前置知识快闪复习段”；  
     - 哪些节点可以合并讲、哪些需要拆开。

3. **查看该节点的 Skills 运用**
   - 在节点详情中找到 **「🎯 Skills 运用」** 区块：  
     - 每个 Skill 后若有 **“含深度内容”** 徽章，表示已经有完整的 Phase2 深度内容（主题 + 练习 + 项目）。  
   - 点击有代表性的 1–2 个 Skill，记下适合课堂使用的：
     - 一两个 **进阶主题**（用于讲解/板书）；  
     - 1–3 道 **进阶练习**（用于课堂问答/板演）；  
     - 0–1 个 **项目任务**（用于课后作业或单元项目）。

4. **参照课例与 Agents 设计整节课的结构**
   - 打开 `limit-conflict-teaching.html`：  
     - 参考“极限认知冲突”两大案例的**三阶段九步结构**，把本节课大致分成：  
       1) 诊断与激活旧知  
       2) 制造与暴露认知冲突  
       3) 模型与方法的精化  
       4) 巩固与迁移
   - 打开 `teacher-agents.html`：  
     - 想一想，本节课主要借助哪些“虚拟助教角色”：  
       - 如需要分析薄弱点 → **Domain Diagnostic Agent**；  
       - 设计冲突情境 → **Cognitive Conflict Designer**；  
       - 布置作业与自动生成题目 → **Assessment & Assignment Agent**。  
     - 这些角色可以直接转化为你在大模型/AI 工具里的提示词模板。

> **课前建议**：  
> 至少选定 **1 个目标节点** + **2–3 个前置节点** + **1–2 个 Skill**，并在纸上或电子教案里写下“本节课的核心变化路径”（从哪里来 → 到哪里去）。

---

### 2. 课中 45–90 分钟：课堂使用方式

#### 2.1 利用主图进行“情境导入与复盘”

- 上课开始 3–5 分钟，在屏幕上展示主图：
  - 让学生看到 **当前知识点在整张图谱中的位置**。  
  - 简要提示：“我们今天所在的学域是……，这个节点前面需要掌握……，后面将通向……。”

#### 2.2 使用认知冲突课例页面

- 若讲授极限或采样定理：
  - 打开 `limit-conflict-teaching.html`，按页面中的流程走一遍：
    - 利用内嵌的 **GeoGebra 动画** 让学生先按直觉猜结论；  
    - 再用严格定义（ε–δ 或频域分析）纠正直觉，形成认知重构。

#### 2.3 在节点详情中调用 Skills 深度内容

- 针对某个关键节点（例如“极限的定义”“级数收敛判别”“常微分方程求解”）：
  1. 在图中点击节点 → 打开节点详情。  
  2. 在「🎯 Skills 运用」里，点击带“含深度内容”徽章的 Skill 的「去运用」。  
  3. 右侧会弹出 Skill 面板，展示：
     - **进阶主题**：可直接用于黑板推导、讲解提纲；  
     - **进阶练习**：适合作为课堂即时问题、小组讨论题；  
     - **项目任务**：可用作后续单元项目或跨课题作业。

- 课堂上可以这样用：
  - 选 1 题练习，让学生先独立思考 2–3 分钟；  
  - 需要时展示“提示”“解答关键步骤”“要点”，引导学生自我修正；  
  - 小结阶段用项目任务的“目标/任务/交付物”部分，说明本章在工程/应用中的意义。

---

### 3. 课后：作业、小测与项目安排

#### 3.1 设计分层作业与小测

- 打开 `test-skills-system.html` → 选择「Phase2 深度内容」标签：
  - 从下拉框中选定某个 Skill，查看其 **所有进阶练习**：  
    - 难度等级、提示、关键步骤、预计时间一目了然；  
    - 可根据学生水平选择基础/进阶/挑战题。
- 将这些题目稍作改写，即可形成：
  - **课后作业清单**（按难度分 A/B/C 组）；  
  - **课堂小测**（5–10 题的纸笔测或在线测验）。

#### 3.2 布置项目化任务

- 依托每个 Skill 的 `projects`，设计：
  - 单元末尾的 **小型建模项目**；  
  - **学习总结型**作业（如“积分技巧小册子”“ODE 求解思维导图”等）。
- 可直接使用项目中给出的：
  - **objectives**：学习目标；  
  - **tasks**：分步任务；  
  - **deliverables**：学生需要产出的成果形式。

---

### 4. 面向学生自主学习与辅导的建议

1. **学生自助入口**：  
   - 把 `student-skills.html` 发给学生或嵌入课程平台，作为“**技能看板**”：  
     - 让学生在预习/复习时，按主题选择对应的 Skill 自主练习。

2. **一对一 / 小组辅导**：  
   - 根据学生当前卡住的知识点，在主图中定位节点；  
   - 结合该节点绑定的 Skill（尤其是“含深度内容”的 Skill），选择合适的题目和项目任务进行针对性辅导。

3. **与 AI 助教结合**：  
   - 在你的 AI 工具中，将“当前节点 + Skill 深度内容 + 学生作答”作为上下文，让 AI 帮你：  
     - 生成同类型变式题；  
     - 给出层级化反馈与补救建议；  
     - 按 `teacher-agents.html` 中的角色（诊断、路径规划、反思等）生成教学分析报告。

## 五、教研与二次开发建议

- **共建题库与项目**：  
  教研组可在 `data/skills-content-phase2.json` 中协同扩展各 Skill 的 `advancedExercises` 与 `projects`，形成本校/本专业的特色题库。

- **扩展更多课例页面**：  
  参考 `limit-conflict-teaching.html`，为导数、积分、级数、概率等主题增设更多“认知冲突课例”页面。

- **引入真实班级数据**：  
  后续可以将实际测验/作业数据与图谱节点、Skills 绑定，为教师侧 Agents 提供更真实的学习分析基础。

---

如需针对某一章（如“导数章”或“级数章”）的**专用教学方案**，可以在此手册的基础上为该章单独写一份「教学场景 + 节次安排 + 对应 Skills/Agents」的细化设计。老师只需按图索骥，即可在不同班级快速复用与调整。
