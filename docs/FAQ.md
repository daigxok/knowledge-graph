### Q5: 数据存储在哪里？

**A**:

## 🔧 功能使用

### Q6: 如何切换语言？

**A**: 
1. 点击右上角的语言切换器 🌐
2. 选择语言：
   - 🇨🇳 中文

### Q7: 如何查看节点的可视化？

**A**: 
1. 点击支持可视化的节点
2. 在详情面板中滚动到"可视化"部分
3. 点击"启动可视化"按钮
4. 调节参数观察变化

**支持可视化的节点类型**:
- 曲率相关节点
- 向量场节点
- 偏微分方程节点
- 优化算法节点
- 概率分布节点

---

### Q8: 如何导出学习路径？

**A**: 
1. 设置好学习路径
2. 点击"导出"按钮
3. 选择格式：
   - PDF: 完整的学习路径文档
   - Markdown: 文本格式
   - PNG: 图谱截图
   - JSON: 数据格式
4. 下载文件

---

### Q9: 如何分享给他人？

**A**: 
1. 设置好图谱视图和过滤条件
2. 点击"分享"按钮
3. 点击"生成链接"
4. 复制链接发送给他人

链接包含：
- 当前视图位置
- 过滤条件
- 选中的节点
- 学习路径

---

### Q10: 如何保存学习进度？

**A**: 
系统自动保存进度到浏览器本地存储。

**手动备份**:
1. 点击"导出数据"
2. 下载 JSON 文件
3. 保存到安全位置

**恢复进度**:
1. 点击"导入数据"
2. 选择之前导出的 JSON 文件
3. 确认导入

## 💻 技术问题

### Q11: 可视化无法显示怎么办？

**A**: 检查以下几点：

1. **浏览器支持**
   - 确保使用支持 WebGL 的现代浏览器
   - 更新到最新版本

2. **硬件加速**
   - 在浏览器设置中启用硬件加速
   - Chrome: 设置 → 高级 → 系统 → 使用硬件加速

3. **网络连接**
   - 确保可以加载 D3.js 和 Plotly.js
   - 检查浏览器控制台是否有错误

4. **清除缓存**
   - 清除浏览器缓存
   - 刷新页面（Ctrl+F5）

---

### Q12: 数据加载很慢怎么办？

**A**: 优化建议：

1. **使用过滤功能**
   - 减少显示的节点数量
   - 按学域或难度筛选

2. **关闭不必要的功能**
   - 暂时关闭可视化
   - 减少同时打开的详情面板

3. **检查网络**
   - 确保网络连接稳定
   - 首次加载后会缓存数据

4. **浏览器优化**
   - 关闭不必要的标签页
   - 清理浏览器缓存

---

### Q13: 移动端性能不好怎么办？

**A**: 移动端优化：

1. **减少节点数量**
   - 使用过滤功能
   - 只显示相关节点

2. **关闭动画**
   - 在设置中关闭过渡动画
   - 减少可视化效果

3. **使用简化模式**
   - 系统会自动检测设备
   - 在低性能设备上使用简化渲染

4. **更新系统**
   - 更新到最新的 iOS/Android 版本
   - 更新浏览器应用

---

### Q14: LaTeX 公式显示不正常？

**A**: 解决方法：

1. **等待加载**
   - MathJax 需要时间加载
   - 等待几秒钟

2. **检查网络**
   - 确保可以访问 MathJax CDN
   - 检查浏览器控制台错误

3. **刷新页面**
   - 按 Ctrl+F5 强制刷新
   - 清除缓存后重新加载

4. **使用离线版本**
   - 下载 MathJax 到本地
   - 修改 HTML 中的引用路径

## 🔧 故障排除

### Q19: 页面无法加载怎么办？

**A**: 故障排除步骤：

1. **检查文件路径**
   ```
   确保文件结构完整：
   - index.html
   - js/
   - data/
   - styles/
   ```

2. **检查浏览器控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 标签的错误信息
   - 查看 Network 标签的加载状态

3. **清除缓存**
   - Ctrl+Shift+Delete 打开清除缓存对话框
   - 选择"缓存的图片和文件"
   - 点击"清除数据"

4. **尝试其他浏览器**
   - 使用 Chrome/Firefox/Safari 测试
   - 确认是否是浏览器兼容性问题

---

### Q20: 数据文件损坏怎么办？

**A**: 数据恢复：

1. **使用备份文件**
   - 查找 `.backup` 后缀的文件
   - 重命名为原文件名

2. **重新生成数据**
   ```bash
   node scripts/generate-all-phase2-nodes.js
   ```

3. **验证数据**
   ```bash
   node scripts/data-validator.js
   ```

4. **从源代码恢复**
   - 从 Git 仓库拉取最新版本
   - 或从发布包中提取数据文件

---

### Q21: 导出功能不工作？

**A**: 导出问题解决：

1. **检查浏览器权限**
   - 允许下载文件
   - 检查下载文件夹权限

2. **检查文件大小**
   - 大文件可能需要更长时间
   - 等待生成完成

3. **尝试不同格式**
   - 如果 PDF 失败，尝试 Markdown
   - 如果 PNG 失败，尝试 JSON

4. **查看控制台错误**
   - 按 F12 打开开发者工具
   - 查看错误信息

---

### Q22: 分享链接无法打开？

**A**: 分享链接问题：

1. **检查链接完整性**
   - 确保复制了完整的 URL
   - 包含所有参数

2. **URL 长度限制**
   - 某些浏览器有 URL 长度限制
   - 尝试减少分享的内容

3. **编码问题**
   - 确保 URL 正确编码
   - 避免特殊字符

4. **使用短链接服务**
   - 使用 bit.ly 等服务缩短链接
   - 或使用导出功能分享文件

---

### Q23: 学习进度丢失了？

**A**: 进度恢复：

1. **检查浏览器存储**
   - 按 F12 打开开发者工具
   - Application → Local Storage
   - 查看是否有数据

2. **恢复备份**
   - 如果之前导出过 JSON 文件
   - 使用"导入数据"功能恢复

3. **避免清除浏览器数据**
   - 清除缓存时不要选择"Cookie和其他网站数据"
   - 或先导出进度再清除

4. **使用云同步**
   - 定期导出进度到云盘
   - 或使用浏览器同步功能

3. **提交报告**

   - 或在 GitHub 创建 Issue

4. **临时解决方案**

   - 尝试故障排除步骤
